# PM Calendar — ข้อกำหนดการใช้งานปฏิทิน

| อัปเดต | 2026-05-07 |
|--------|------------|

เอกสารนี้เก็บ **ข้อกำหนดผลิตภัณฑ์** สำหรับมุมมองปฏิทิน F02 — อ้างอิงความหมายคอลัมน์ **`System status`** ใน [`PM_SCHEDULING_SHEET_REFERENCE.md`](PM_SCHEDULING_SHEET_REFERENCE.md) (Sheet 2)

---

## 1. มุมมองปฏิทิน (Calendar view)

ระบบต้องรองรับการ **จัดการและวางแผนงาน PM** ในรูปแบบ **ปฏิทิน** (เช่น Month / Week / Day) เพื่อดูและปรับช่วงเวลาตามแผน

---

## 2. การวางแผนและกรองข้อมูล (รายบุคคล / มิติธุรกิจ)

ผู้ใช้ต้องสามารถ **จัดลำดับความสำคัญของการวางแผนแบบแยกมุมมอง** ได้โดยเลือก **ประเภทการกรอง** เช่น

- กรองตาม **ไลน์การผลิต** / สถานที่ (Functional Location หรือข้อมูลที่เทียบเท่าในระบบ)
- กรองตาม **ชื่อช่าง / ทรัพยากรที่ได้รับมอบหมาย** (Assigned resources / Work center)
- กรองตาม **สถานะใบงาน**, **ประเภทใบงาน**, **ความเร่งด่วน** และตัวเลือกอื่นที่ทีมกำหนดในเมนูกรอง

*(รายการคอลัมน์กรองที่ implement ในแอปควรสอดคล้องกับ API และการตั้งค่า filter menu ตามบทบาทผู้ใช้)*

---

## 3. สีของแต่ละ Work Order ตาม System Status (SAP)

สีบนปฏิทินแสดงตาม **ข้อความ System status** จาก SAP (ใช้หลักการ token / prefix 4 ตัวอักษรแรกตาม Sheet 2 เมื่อจำเป็น)

| สี | ความหมาย |
|----|-----------|
| **เขียว** | Work Order มีสถานะ **TECO** หรือ **CLSD** — งานถูกปิดแล้ว (Technically complete / Closed) |
| **แดง** | Work Order **มีเลข WO ในระบบ** แต่สถานะ **ไม่ใช่** TECO/CLSD — งานยังไม่ถูกปิด (เช่น **REL**, **CRTD** หรือสถานะอื่นที่ไม่ใช่ปิดงาน) |
| **ฟ้า** | **แผนงานที่ยังไม่ถูก generate เลข Work Order** (ยังไม่มีเลข WO ที่ถือว่าใช้งานได้ในระบบ — เทียบกับ metadata / `order_number` ตามกติกาในแอป) |

**หมายเหตุ:** ไม่ใช้สีเพื่อแทน “เลยกำหนดแผนแล้วหรือยัง” โดยแยกจากสถานะ SAP — ตาม requirement นี้การแบ่งสีหลักผูกกับ **System Status** และความมีอยู่ของ **เลข WO** ตามตารางด้านบน

---

## 4. การแสดงหลายแผนในแต่ละวัน (มุมมอง Month)

ใน **มุมมองเดือน (Month)** แต่ละวันต้องแสดง **จำนวนแผนงาน / ใบงานที่ทับซ้อนวันนั้น** เป็นรายการ **เรียงซ้อนลงมา** (full-width blocks ในเซลล์วัน) — **ไม่แสดงแกนเวลารายชั่วโมง** และ **ไม่ระบุช่วงเวลา (clock time) บนตัวบล็อกของแต่ละแผน** (ไม่ใช่ Gantt แบบมีช่วงเริ่ม–จบภายในวัน)

- ข้อความบนบล็อกใช้ระบุตัวแผนงาน (เช่น เลขอ้างอิง / คำอธิบายหัวข้อ) — ตามตัวอย่างจากลูกค้า  
- ถ้าต้องการดู **ช่วง planned start/finish** หรือเวลาโดยละเอียด ใช้ **Popover (ชี้เมาส์)** / **Modal รายละเอียด (ดับเบิลคลิก — ข้อ 5)** หรือมุมมอง **Week / Day** (Day ใช้แกนชั่วโมงตามการออกแบบแยกต่างหาก)

---

## 5. ดูรายละเอียดแผนงาน (Double-click → Pop-up)

เมื่อต้องการทราบ **รายละเอียดย่อยของแผนงาน / ใบงาน** บนปฏิทิน ผู้ใช้สามารถ **ดับเบิลคลิก** ที่แท่งงาน — ระบบเปิด **หน้าต่างป๊อปอัป (Modal)** แสดงข้อมูลและแท็บที่เกี่ยวข้อง

- **คลิกครั้งเดียว** ไม่ใช้เปิด Modal (ลดการชนกับการลากย้ายแผน — Drag & Drop)  
- **ชี้เมาส์ (hover)** ยังใช้แสดง Popover สรุปได้ตามเดิม

### 5.1 แท็บ «Task»

- แสดง **รายละเอียดของแผนงาน** แบบอ่านเป็นหลัก ได้แก่  
  - **Maintenance Plan** — เลขอ้างอิงแผน / ใบงานในระบบ (ตามข้อมูลที่ normalize)  
  - **PM Task detail** — ข้อความและรายการที่สอด **ชีต Excel «PM Task List»** ของลูกค้า (ดู [`PM_TASK_LIST_SHEET_REFERENCE.md`](PM_TASK_LIST_SHEET_REFERENCE.md))  
- แอปดึงข้อความ PM Task detail จาก `work_orders.ui_metadata_json` ตามคีย์ที่ระบุในเอกสาร PM Task List — ถ้ายังไม่มีข้อมูลจากการนำเข้า ให้แสดงสถานะว่าไม่มีรายการ  
- (ทางเลือกตาม scope) แนบรูป / เอกสารประกอบงานในแท็บนี้

### 5.2 แท็บ «Planning»

- ให้ผู้มีสิทธิ์แก้ไขสามารถ **วางแผนผู้ปฏิบัติงาน** (ทรัพยากร / ศูนย์งานตามที่ออกแบบในระบบ — เทียบเท่า Plan Work Center ใน UI เดิมของลูกค้า)  
- ให้สามารถ **วางแผนเวลาที่จะทำงาน** — ช่วง **เริ่ม** และ **สิ้นสุด** (Plan time) แล้วบันทึกลง `planned_start` / `planned_finish`  
- ปุ่ม **Save** / **Don't save** / **Cancel** ตาม workflow ที่ทีมกำหนด

### 5.3 แท็บอื่น

- แท็บ **Close Work Order** ใช้บันทึกการปิดงานจริง — แยกจากแท็บ Planning; การเลือกวัน–เวลาและไอคอนนาฬิกา → **§7**

---

## 6. Drag & Drop — ย้ายแผนไปวันอื่น

ผู้ใช้สามารถ **ลากวาง (Drag & Drop)** แผนงานจากวันหนึ่งไปวันใหม่ได้ในมุมมองที่รองรับ (เช่น Week / Month) โดยมีเงื่อนไขดังนี้

| สีแท่งงาน | ย้ายได้ | ก่อน Drop (หน้าต่างสาเหตุ) |
|-----------|---------|------------------------------|
| **แดง** (มี WO ยังไม่ปิด) | ได้ | ต้องเลือก **Reason Code** จากรายการมาตรฐาน — **ไม่เลือกจะ Save ไม่ได้**; ใส่ **Comment** เพิ่มได้ (และอาจบังคับความยาวเมื่อเลือกรหัสเฉพาะ เช่น “อื่นๆ”) |
| **ฟ้า** (ยังไม่มีเลข WO — มักต้องมี **Call no.** ใน metadata จึงย้ายได้ตามกติกาเทคนิค) | ได้ตามกติกา | เปิดหน้าต่างถามสาเหตุเช่นกัน แต่ **Reason Code ไม่บังคับ** — Save ได้แม้ไม่เลือก; Comment ได้ |
| **เขียว** (TECO / CLSD) | **ไม่ได้** | — |

รายการ **Standard Reason Code** (เช่น 01–05) ใช้ร่วมกันทั้งแดงและฟ้าเมื่อผู้ใช้เลือกกรอก

---

## 7. การปิดงาน (Close WO) จากปฏิทิน

ผู้ใช้สามารถเปิดใบงานจาก **ปฏิทิน** แล้วบันทึกการปิดงานจริงได้โดยตรง:

- เปิดหน้าต่างรายละเอียดจากแท่งงานบนปฏิทิน (เช่น **ดับเบิลคลิก** ตาม §5) แล้วไปที่แท็บ **Close Work Order**  
- ในฟอร์มปิดงาน ผู้ใช้ต้องเลือก **วันที่และเวลาเริ่ม / วันที่และเวลาเสร็จ** — ใช้ตัวเลือกวันที่ (ปฏิทิน) และ **แผงเลือกเวลา** ในคอมโพเนนต์เดียวกัน โดยมี **ไอคอนนาฬิกา** เป็นจุดสังเกตการเลือกเวลา (เทียบ UI เดิมที่คลิกจากรูปนาฬิกา)  
- การปิดงานแยกจากแท็บ **Planning** (แผนล่วงหน้า) — Planning ใช้ช่องวัน–เวลาแบบเดียวกันเพื่อความสม่ำเสมอ
