## ภาพรวมระบบ — flow เดียว

**ใช้เล่า “ทั้งระบบเป็นสายเดียว” ตั้งแต่ข้อมูลเข้าจนถึงผู้บริหารดูตัวเลข**

**ภาษาไทย**

**ไฟล์จาก SAP** `>` **ผู้ใช้เข้าสู่ระบบ** `>` **นำเข้าไฟล์เข้าแอป** `>` **ให้ระบบประมวลผลเป็นใบงานในระบบ** `>` **ช่างเปิดใบงานและแนบหลักฐานรูป** `>` **หัวหน้าดูแดชบอร์ดและรายงาน KPI** `>` *(รอบถัดไป: นำเข้าใหม่ / อัปเดต KPI ตามวัน)*

**English (same overview)**

**SAP export (file)** `>` **Login** `>` **Import into the app** `>` **System builds/updates work orders** `>` **Technician opens a work order and attaches evidence photos** `>` **Supervisor reviews Dashboard & KPI reports** `>` *(next cycle: new import / refresh KPI as needed)*

## Flow ตามบทบาท — ชื่อไทย

### ช่าง (Technician)

**เส้นทางหลักที่ต้องการ (ตามแนวที่คุณยกตัวอย่าง):**

**เข้าสู่ระบบ** `>` **แดชบอร์ด** `>` ***แดชบอร์ดส่วนตัว (งานของฉัน)*** `>` **เลือกใบงาน** `>` **แนบหลักฐานรูปก่อน–หลังทำ** `>` **บันทึกเสร็จ**

**อธิบายสั้น ๆ**

- ตอนนี้ระบบรองรับให้ช่าง **เข้าสู่ระบบ** แล้วไป **แดชบอร์ด** เพื่อดูภาพรวม และไป **หลักฐานรูป** เพื่อแนบรูปกับใบงานได้ชัดเจน  
- หน้า **“แดชบอร์ดส่วนตัว — เฉพาะงานที่มอบหมายให้ฉัน”** เป็นเป้าหมายตาม requirement — *เมื่อพัฒนาครบ จะอยู่หลังขั้นแดชบอร์ดแบบเลือกมุมมอง*  
- ถ้ายังไม่มีมุมมองส่วนตัว ช่างยังใช้ **รายการใบงาน** + **เปิดใบงานที่รู้หมายเลข** แล้วไป **หลักฐานรูป** แทนได้

**เส้นทางสำรอง (ใช้ได้กับของที่มีอยู่แล้ว):**

**เข้าสู่ระบบ** `>` **ใบงาน** `>` **เปิดใบงานที่ต้องทำ** `>` **ไปหลักฐานรูป (ระบบช่วยใส่เลขใบงานให้)** `>` **แนบรูป** `>` **จบ**

---

### ผู้วางแผน / ผู้ดูแลข้อมูล (Planner)

**เข้าสู่ระบบ** `>` **นำเข้าข้อมูล SAP** `>` **เลือกไฟล์ (IW37N / Confirm / GI / GR)** `>` **อัปโหลด** `>` **กดให้ระบบประมวลผลเข้าตารางงาน** `>` **ตรวจว่านำเข้าสำเร็จหรือมีข้อผิดพลาด** `>` **ดูรายการใบงาน** `>` **เปิดรายละเอียดใบงานเมื่อต้องตรวจ**

**อธิบายสั้น ๆ**

- ขั้น “ประมวลผลเข้าตารางงาน” คือให้ข้อมูลจากไฟล์กลายเป็น **ใบงานในระบบ** ที่ช่างและหัวหน้าใช้ต่อได้  

---

### หัวหน้างาน / ผู้จัดการ (Supervisor)

**เข้าสู่ระบบ** `>` **แดชบอร์ด** `>` **ดูตัวเลขภาพรวม (งานค้าง / นำเข้าล่าสุด / แนวโน้ม KPI)** `>` **รายงาน KPI (ละเอียด)** `>` **ตัดสินใจ / มอบหมายต่อ (นอกระบบหรือในระบบตามแผน)**

**ถ้าต้องการให้ตัวเลข KPI อัปเดตตามวันที่:**

**แดชบอร์ด** `>` **สั่งอัปเดตตัวเลข KPI ประจำวัน** `>` **รอระบบประมวลผล** `>` **กลับมาดูกราฟ / รายงาน KPI อีกครั้ง**

---

### ผู้ดูแลระบบแอป (Administrator)

**เส้นทางหลักที่ต้องการ (ตามแผน F10 — RBAC / จัดการผู้ใช้):**

**เข้าสู่ระบบ** `>` ***เมนูจัดการผู้ใช้ / บทบาท (หน้าแอดมิน)*** `>` **ค้นหาหรือเลือกผู้ใช้** `>` **สร้างผู้ใช้ใหม่หรือแก้ข้อมูลผู้ใช้** `>` **กำหนดบทบาท (เช่น Planner / ช่าง / ผู้ดูอย่างเดียว / ผู้ดูแลระบบ)** `>` **บันทึก** `>` **แจ้งผู้ใช้ทดสอบล็อกอินและเมนูตามสิทธิ**

**อธิบายสั้น ๆ**

- บทบาทนี้ดูแลว่า **ใครเข้าแอปได้** และ **เห็นเมนู–ทำ action อะไรได้** ให้ตรงนโยบายองค์กร — แยกจากงานวางแผนซ่อม (Planner) หรือหัวหน้าดู KPI  
- ในแผนเอกสารโครงการ หน้าจัดการผู้ใช้อยู่ที่เส้นทางประมาณ **`/admin/users`** — *ถ้ายังไม่มีในเวอร์ชันที่ deploy ให้ดูแลผู้ใช้ผ่านทีม IT / ฐานข้อมูลชั่วคราวตามที่องค์กรกำหนด*  
- **ผู้ดูแลโครงสร้างเซิร์ฟเวอร์** (Docker, ฐานข้อมูล, worker, เครือข่าย) เป็นงาน **IT / Dev** ตาม SOP ติดตั้ง — ไม่ใช่ปุ่มในแอปเดียวกับ flow ด้านบน แต่ต้องทำก่อนแอปใช้งานจริงได้

## Flow ตามบทบาท — English role titles

*Same journeys as above; step labels in English for stakeholder decks / bilingual packs.*

### Technician

**Target path**

**Login** `>` **Dashboard** `>` ***Personal dashboard (my jobs)*** `>` **Open a work order** `>` **Attach before/after photos** `>` **Save / done**

**Fallback (works with current screens)**

**Login** `>` **Work orders** `>` **Open the job** `>` **Evidence upload (work order ID prefilled)** `>` **Upload photos** `>` **Done**

---

### Planner

**Login** `>` **SAP import** `>` **Pick file type (IW37N / Confirm / GI / GR)** `>` **Upload** `>` **Run processing into work orders** `>` **Check success or errors** `>` **Browse work orders** `>` **Open work order details if needed**

---

### Manager

**Login** `>` **Dashboard** `>` **Review summary (backlog / recent imports / KPI trend)** `>` **KPI report (detail)** `>` **Decide & assign (process / future in-app assignment)**

**Optional KPI refresh**

**Dashboard** `>` **Request daily KPI snapshot** `>` **Wait for background processing** `>` **Review charts / KPI report again**

---

### Administrator (application admin)

**Target path (planned F10 — RBAC / user management)**

**Login** `>` ***Admin: users & roles screen*** `>` **Find or select a user** `>` **Create or edit user** `>` **Assign roles (e.g. Planner / Technician / read-only viewer / admin)** `>` **Save** `>` **Ask the user to log in and verify menus vs. permissions**

**Notes**

- This role owns **who can sign in** and **what they can see or do** — separate from day-to-day planning or KPI supervision.  
- Project docs target a route such as **`/admin/users`**. *If that screen is not on your build yet, user provisioning may go through IT / DB per org policy.*  
- **Server / infra operations** (Docker, database, worker, network) follow **IT / Dev SOPs** — not the same in-app clicks as above, but required before production use.

## เส้นทางที่ทุกคนใช้ร่วมกัน

1. **เปิดเว็บแอป** `>` **เข้าสู่ระบบ** (ใส่รหัสผู้ใช้ / รหัสผ่านตามที่องค์กรกำหนด)  
2. ถ้าโหมดทดสอบพิเศษ อาจข้ามล็อกอินได้ — ตามที่ IT แจ้ง  
3. ใช้ **เมนูด้านข้าง** เลือกไป **แดชบอร์ด / ใบงาน / นำเข้า SAP / หลักฐานรูป / รายงาน** ตามบทบาท — *ผู้ดูแลระบบแอป: **จัดการผู้ใช้ / บทบาท** ตามแผน F10*  

**English**

1. **Open the web app** `>` **Login** (org username / password)  
2. Special test mode may skip login — per IT  
3. Use the **side menu** for **Dashboard / Work orders / SAP import / Evidence / Reports** by role — *application admin: **Users & roles** per F10 plan*

## สรุปว่า “จบ flow” เมื่อไร

| บทบาท (ไทย) | Role (EN) | จบ flow เมื่อ |
|--------------|-----------|----------------|
| ช่าง | Technician | แนบหลักฐานรูปกับใบงานครบตามขั้นตอนงาน (ก่อน–หลัง) และบันทึกสำเร็จ |
| ผู้วางแผน | Planner | ไฟล์ SAP เข้าระบบ ไม่มี error สำคัญ และใบงานโผล่ในรายการให้ทีมใช้งาน |
| หัวหน้า | Supervisor | อ่านแดชบอร์ด / รายงาน KPI ได้ตามรอบที่ต้องการ |
| ผู้ดูแลระบบแอป | Administrator | ผู้ใช้และบทบาทในระบบตรงกับนโยบายองค์กร และผู้ใช้ที่ตั้งค่าใหม่ล็อกอิน–ใช้เมนูได้ตามสิทธิ (เมื่อมีหน้าจัดการผู้ใช้ครบตามแผน) |

## ขอบเขตเวอร์ชันปัจจุบัน (พูดตรง ๆ กับลูกค้า)

- **มีแล้ว:** เข้าสู่ระบบ, แดชบอร์ดภาพรวม, นำเข้า SAP, ประมวลผลเป็นใบงาน, ดูใบงาน + รายละเอียด, แนบหลักฐานรูป, รายงาน KPI, ติดตามงานหลังบ้านเมื่อสั่งประมวลผลแบบคิว  
- **กำลังตามแผน requirement:** **หน้าผู้ดูแลระบบ (จัดการผู้ใช้–บทบาท / RBAC, เช่น `/admin/users`)**, ปฏิทินงาน, ลากวางงาน, มอบหมายช่างบนจอ, **แดชบอร์ดส่วนตัวเฉพาะงานของฉัน**, ยืนยันใบงานในแอปแบบเต็ม flow, ส่งออกรายงาน SAP จากแอป  

รายการ “แผน” ไม่ได้แปลว่าระบบใช้ไม่ได้ — แปลว่า **เส้นทางแบบ “Personal Dashboard” อาจยังเป็นขั้นถัดไป** หลังภาพรวมและหลักฐานรูปนิ่งแล้ว
